# 111: Three beauty rounds, then ship

## Goal

Final polish campaign: three rounds focused purely on code beauty (simplicity, elegance, structure, naming, idiom consistency) per CLAUDE.md's code style. Behavior must not change: generated JSON byte-identical, prerendered HTML identical modulo asset hashes and the router build timestamp. After round 3: commit, push, deploy to prod (operator-authorized).

## Beauty doctrine (from CLAUDE.md, enforced)

Less is more. No comments. No single-use helpers, inline the expression. Happy path only, positive conditionals, no failure-first guards. Ternary for simple conditionals. Default params over null checks. Small files, predictable placement. One component per file. Views presentational, hooks own logic, routes thin. Type-safe end to end. Beyond the letter: consistent idioms for the same shape of problem across files, names that make comments unnecessary, no over-parameterization, no residual scar tissue from refactors.

## Protocol

Per round: parallel per-area artisans with disjoint file sets -> each proves byte-neutrality (build + diff vs /tmp/mb-beauty-baseline, HTML via checksum list modulo hashes/timestamp) -> lead gates (tsc, build, contracts). Round 2 = fresh eyes on the post-round-1 tree, stricter. Round 3 = until-dry final pass, then tester smoke, scribe commit, push, deployer.
