# 19 — Clean titles for vacuous Plenartitel

## Goal

Many Handzeichen votes have official Plenartitel that look like `Entschließungsantrag der AfD zu Drucksache 21/5808` — they repeat the proposing Fraktion (already shown as Antragsteller in the UI) and reference a Drucksache number instead of saying what the vote is *about*. Namentliche votes are usually fine because clerks craft them with a real subject.

Add an LLM-driven cleanup pass that produces a short neutral subject title and store it alongside the original. UI prefers the cleaned title when available; the official Plenartitel is preserved verbatim because it is the source-of-truth document title and we never mutate it.

## Contracts

### Schema

Add to `db/schema/votes.ts`:

```ts
cleanTitle: text('clean_title'),
```

Nullable. Null means "no rewrite happened" → UI falls back to `title`. Migration in `db/migrations/`.

### Selection (which votes get a rewrite attempt)

Only run on rows matching ALL:

- `procedural = 0`
- `clean_title IS NULL` (idempotent; `--force` flag re-runs)
- Title matches vacuous-shape regex. Start with these patterns, expand as we see failures:
  - `^(Antrag|Entschließungsantrag|Gesetzentwurf|Beschlussempfehlung)\s+(der|des)\s+(Fraktion|Abgeordneten|Bundesregierung)`
  - `zu Drucksache\s+\d+/\d+\s*$`
  - Title length under ~30 chars with no nouns past the doc-type

Confirm regex against actual title distribution before running at scale — sample a few hundred rows and eyeball.

### LLM call

`claude -p --model sonnet --output-format json`, pLimit 4, prompt on stdin.

Inputs available per vote:
- `title` (official Plenartitel)
- `summary_simplified` from `vote_descriptions` (strong signal — already a clean human-readable summary)
- Underlying Drucksache title from `drucksachen` cache when extractable from `document` field (reuse `extractDrucksachen` + `readDrucksacheCache` from `etl/bundestag/polarity/rule.mjs`)

Output JSON:

```json
{ "clean_title": "string|null", "confidence": "high|medium|low" }
```

Rules baked into the prompt:
- ≤80 characters, starts with a capital letter
- No Fraktion names (CDU/CSU, SPD, AfD, Grüne, Linke, FDP, BSW, etc.) — proposer is shown separately
- No `Drucksache NN/NNNN` references
- No procedural framing (`Antrag der ...`, `Beschlussempfehlung zu ...`) — name the subject directly
- Neutral, allgemeinverständlich, no parteiliche Bewertung
- No em/en dashes; comma, colon, or new sentence instead (see CLAUDE.md style rules)
- If unclear or the official title is already good, return `clean_title: null` with `confidence: high`

Write `clean_title` only when `confidence` is `high` or `medium` AND the value is non-null. Low-confidence → leave NULL.

### Read path

`apps/bundestag/src/server/votes.ts` (list query + detail query) selects both `title` and `cleanTitle`. Types exported to frontend include both.

Display rule: heading uses `cleanTitle ?? title`. On detail page, if `cleanTitle` is set, render the official `title` as a smaller secondary line under the heading (label: `Offizieller Titel`) so the source remains visible.

## Open questions

- Should we also run this for namentliche votes that happen to match the vacuous-shape regex? Default: yes, regex is the gate, not vote_type. Cheap to skip via filter if it produces noise.
- Re-run cadence: title cleanup is content-derived, so re-running after `summary_simplified` improves is worth it. Add a `--force-since <date>` flag in run.mjs for later iteration.
