# 111: Multi-parliament MVP (EU + Berlin + Bayern), web only

## Goal
Extend the web app to three new parliaments behind path prefixes: /eu/ (European Parliament), /berlin/ (Abgeordnetenhaus Berlin), /bayern/ (Bayerischer Landtag). One mega app, clean separation: NOTHING in the existing Bundestag tables, ETL, routes, or server fns may change behavior. Web only (no iOS). Verify on the local dev server (vite :5174 -> dev.machtblick.de). DO NOT push, DO NOT deploy.

## Architecture decisions (lead, fixed)
- One web app (apps/bundestag) with path-prefixed sections. Subdomains (eu.machtblick.de) come later as rewrites; not part of this MVP.
- New parliament-scoped DB tables with a `parliament` discriminator ('eu' | 'be' | 'by'). Do NOT touch existing bundestag tables. Prefix: `mp_` (multi-parliament): mp_votes, mp_member_votes, mp_members, mp_parties (plumber may adjust, document here).
- Two ETL sources: etl/europarl/ (official EP Open Data API + HowTheyVote CSV, see plans/110-europarl.md findings) and etl/abgeordnetenwatch/ (v2 API, parameterized by parliament, Berlin + Bayern, see plans/109-landtage.md findings).
- MVP scope per parliament: votes feed, vote detail (hemicycle + per-party/group breakdown + defectors), members grid (photos where licensable), member detail (ballot history), parties/groups overview + detail (seats, cohesion, attendance). NO speeches/Reden tab, NO motions section in MVP.
- EU German lens: mp_members carries nationalParty + country so a "Deutschland" filter on members and a German-party donut row on vote detail are possible; frontend implements the members filter at minimum.
- Reuse presentational components (hemicycle, cards, donuts, stat bars) where they are truly presentational; new server fns + routes per section; follow the existing prerender rules (add new paths to prerenderPaths()).
- Follow whatever DB engine the repo actually uses (inspect db/ + etl/ first); document the truth here.

## Architecture decisions (plumber, verified 2026-07-10)
- **Engine is SQLite via better-sqlite3**, NOT Postgres (CLAUDE.md is stale on this). Single DB file `db/machtblick.sqlite` (WAL). Web app reads it through Drizzle: `db/client.ts` opens `MACHTBLICK_DB ?? db/machtblick.sqlite`, `apps/bundestag/src/server/*` imports `db` from `@machtblick/db/client` and tables from `@machtblick/db/schema`.
- **Schema is dual-defined**: Drizzle TS files in `db/schema/*.ts` (re-exported from `index.ts`) are the source of truth the web app queries; `.mjs` ETL workers create/query the same tables via raw `better-sqlite3` + an idempotent `ensureSchema()` DDL. mp_ tables follow BOTH: new Drizzle files `db/schema/mp*.ts` (for the web agent) AND `etl/europarl/schema.mjs` DDL (operational path). They are kept byte-for-byte consistent (snake_case).
- **Do NOT run `drizzle-kit generate`** — it drifts (wants to DROP `members.dip_person_id`, see plan 06 / plumber memory). mp_ migration `db/migrations/0029_mp_multi_parliament.sql` is hand-written for reproducibility; the live tables were created by `ensureSchema()` during ingest, which is the operational path. `_journal.json` was left untouched.
- ETL convention followed: one folder per source (`etl/europarl/`), Node `.mjs` workers, upsert-by-delete-then-insert scoped to `parliament='eu'` (idempotent, never touches be/by or bundestag tables).

## Contracts (fill in as built)
- **mp_* table shapes (plumber, built):** all carry a `parliament` text discriminator ('eu'|'be'|'by'); all `mp_*` rows for EU use `'eu'`. Web agent: always filter by `parliament`.
  - `mp_parties` PK(parliament, slug): `name, short_name, color?(nullable), seats?, member_count?`. For EU, slug ∈ {evp, sd, pfe, ekr, renew, gruene-efa, linke, esn, ni, id}; `short_name` = EVP/S&D/PfE/EKR/Renew/Grüne-EFA/Linke/ESN/NI; `color` = muted group brand hex.
  - `mp_members` PK(parliament, id): `name, first_name?, last_name?, party(=EU group slug, current), national_party?(EU: e.g. CDU/CSU/AfD/SPD — the German-lens field), country?(ISO3, e.g. DEU), state?(null for EU), picture_url, picture_license`. Indexed on (parliament, party) and (parliament, country).
  - `mp_votes` PK(parliament, id): `date(YYYY-MM-DD), title(EN from HowTheyVote), title_de?(NULL for now — see note), description?, reference?, procedure_reference?, result('angenommen'|'abgelehnt'), yes, no, abstain, absent, total_members, source_url(howtheyvote.eu/votes/{id})`. Indexed (parliament, date).
  - `mp_member_votes` PK(parliament, vote_id, member_id): `choice('ja'|'nein'|'enthalten'|'nicht_abgegeben')`. Indexed (parliament, member_id).
  - `mp_vote_party_summaries` PK(parliament, vote_id, party): `position('yes'|'no'|'abstain'|'mixed'), yes, no, abstain, absent, members`. `party` = EU group slug **at vote time**. Header/summary/ballot counts reconcile exactly (0 mismatches verified).
- **title_de is NULL for EU** in MVP. German vote titles need either an LLM translation step (like `etl/bundestag/translations`) or an EP `vote-results` join keyed by meeting+decision; both deferred. Web agent: fall back to `title` (EN) until a translation ETL lands.
- **mp_parties.seats/member_count = distinct term-10 MEPs per group who cast an is_main ballot** (742 total incl. mid-term replacements), so group totals run slightly above the live 720 (EVP 194 vs ~188 live). Treat as roster size, not live seat count.
- **Landtage (be/by) specifics (plumber, built):** source = abgeordnetenwatch v2 API (CC0), worker `etl/abgeordnetenwatch/` (`npm run etl:landtage`), same delete-then-insert idempotency scoped to `parliament in ('by','be')`.
  - `mp_members.id` = aw **politician id** (stable across terms). `first_name/last_name/national_party/country/state` NULL (API only gives a combined label). `picture_url/picture_license` NULL — aw serves no photos; a Wikidata P18 → Commons step is deferred. Web agent: render initials fallback.
  - `mp_votes.title` = `title_de` = aw poll label (already German). `description/reference/procedure_reference` NULL. `source_url` = the aw poll page (e.g. abgeordnetenwatch.de/bayern/19/abstimmungen/…). `result` from `field_accepted`, fallback yes>no.
  - Party slugs: by = {csu, freie-waehler, afd, gruene, spd}; be = {cdu, spd, gruene, linke, afd, fdp, fraktionslos}. `mp_parties.color` = brand hexes (CSU #0088c9, CDU #1c2a3a, SPD #e3000f, Grüne #1aa037, AfD #009ee0, Linke #be3075, FDP #ffcc00, FW #f39200, fraktionslos #7a7a7a). Summaries key on fraction **at vote time**.
  - **Berlin is a two-term mix**: current period 133 (2021-2026) has only 3 polls, so previous period 107 (2016-2021, 9 polls) is auto-included (rule: current < 10 polls). FDP rows come from the old term. `mp_parties.seats` for be = cross-term roster union, not a live seat count. Bayern = current period 149 only (64 polls).
  - Roster convention matches EU: departed members who cast ballots are resolved on demand and included, so seats run slightly above live totals (by: 207 members vs 203 live).
- **Server fns (web, built)** in `apps/bundestag/src/server/mp*.ts`, created via `createServerFn`, called ONLY from route loaders (never useQuery):
  - `mpVotesList(dbKey)` → `MpVoteListItem[]` (`mpVotes.ts`, exports `MpVoteListItem`, `MpPartySummary`). yes/no/abstain/absent/totalMembers + per-group summaries (label/color/position). `titleIsEnglish` = `!titleDe`.
  - `mpVoteDetail({ parliament, id })` → `MpVoteDetail` (`mpVoteDetail.ts`). Group summaries + defectors (ballot ≠ group majority; group = current `mp_members.party`, approximate historically — no affiliation history in mp_ yet).
  - `mpMembersList(dbKey)` → `MpMemberListItem[]` (`mpMembers.ts`). Per-member attendance + loyalty (share matching group majority) over all ballots; carries nationalParty/country/state for the German lens.
  - `mpMemberDetail({ parliament, id })` → `MpMemberDetail` (`mpMemberDetail.ts`). Member + full ballot history (title/date/choice/result, date desc).
  - `mpPartiesList(dbKey)` / `mpPartyDetail({ parliament, slug })` → `MpPartyListItem[]` / `MpPartyDetail` (`mpParties.ts`). Seats (roster) + cohesion (avg majority share) + attendance; detail adds member roster. Reuses bundestag `partyStats.ts`.
  - Shared: `mpPartyMap.ts` `loadMpParties(dbKey)` → Map slug→{label,color,seats}. Config: `src/lib/parliaments.ts` (`PARLIAMENTS`, `parliamentBySlug`; slug eu/berlin/bayern ↔ dbKey eu/be/by).
- **Route tree (built, DRY — one dynamic tree, not three copies):** `src/routes/$section/{index,votes/$id,members/index,members/$id,parties/index,parties/$slug}.tsx`. Each loader guards `parliamentBySlug(params.section)` → `notFound()`; static top-level routes still win over `$section`. Yields /eu/, /eu/votes/$id, /eu/members/, /eu/members/$id, /eu/parties/, /eu/parties/$slug and the same under /berlin/ and /bayern/. All enumerated in `vite.config.ts > prerenderPaths()` (mp ids per parliament). Views `src/views/mp/*` reuse bundestag presentational primitives 1:1 (VoteHemicycle, VoteDistributionDonut, Stamp) + card language; feed uses a lazy card (eager first 12) so EU 580-vote page is ~3.8MB not 121MB. German lens on /eu/members/ (Deutschland country='DEU' + national_party sub-filter; breakdowns stay by EU group). Parliament switcher + section tabs in `MpSectionNav`.
