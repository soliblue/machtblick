### Round 8 binary-byte verdict (report only)

Every page carries literal NUL bytes (0x00) inside the TanStack Start dehydrated-state inline script, e.g. `i:"__root__\0"` and `i:"\0votes\0$id\0..."` (vote page offsets ~134k and ~288k). TanStack router-core builds route/match cache keys with `"\0"` separators (`node_modules/@tanstack/router-core/dist/esm/path.js:49`: `base + "\0" + to`; also `new-process-route-tree.js:344,354`) and the seroval serialization writes them raw instead of escaping them as \u0000. Framework behavior, not app code or DB data; harmless to browsers but makes grep classify pages as binary. Workaround stands: use node regex for HTML assertions.
